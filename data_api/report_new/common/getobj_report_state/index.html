    Получение текущего состояния

    Получение текущего состояния > Получение текущего состояния
    Метод getobj.report_state
    Описание Получение текущего состояния отчета
    Кому доступен Агент, Клиент
    Необходимые компоненты для работы с методом Data API для настройки
    Вес метода 1 балл

    Параметры запроса

    Название Тип Обязательный Допустимые значения Описание
    access_token string да Ключ сессии аутентификации
    user_id number нет Уникальный идентификатор пользователя клиента агента от имени которого делается запрос
    Является обязательным для агента
    Для получения списка пользователей клиента необходимо использовать метод "get.customer_users"
    report_id number да Уникальный идентификатор отчета пользователя. Для получения списка отчетов пользователей необходимо использовать метод "get.reports_list"

    Параметры ответа

    Название Тип Допустимые значения Ответ по умолчанию Описание
    limit number Нельзя запросить более 200 строк. да Количество возвращаемых записей. По умолчанию 50.
    offset number да Сдвиг - определяет с какого номера записи возвращать limit записей. По умолчанию 0.
    date_from iso8601 YYYY-MM-DD hh:mm:ss Дата начала основной выборки(ограничение 366 дней)
    date_till iso8601 YYYY-MM-DD hh:mm:ss Дата окончания основной выборки
    compared_date_from iso8601 YYYY-MM-DD hh:mm:ss Дата начала сравниваемой выборки(ограничение 366 дней).
    compared_date_till iso8601 YYYY-MM-DD hh:mm:ss Дата окончания сравниваемой выборки.
    perspective_window number да Можно задать период от 1 до 180 дней от сессии до обращения Кол-во дней от сессии до обращения в которых происходит атрибутирование обращения к сессии.
    dimensions array Список используемых измерений. Доступный список измерений см.раздел "Получение вложенности измерений" . Порядок влияет на построение дерева.
    columns array Список используемых столбцов. Доступный список пользовательских столбцов см.раздел "Получение вложенности столбцов" . Порядок влияет на построение отчета
    Фильтры применяемые в отчете
    filter object Фильтры применяемые в отчете. Логика построения аналогична "Критерии фильтрации"
    filters array Выражение, может содержать в себе как простые фильтры, так и дерево фильтров
    field string Поле сущности к которой будет применяться фильтрация (список заранее определён для метода
    operator enum Оператор фильтрации. Список всех операторов можно получить в разделе "Операторы фильтрации"
    value string Значение для оператора фильтрации. Необязательное поле, если оно отсутствует, то считается пустота.
    Быстрые фильтры применяемые в отчете
    rapid_filter object Быстрые фильтры применяемые в отчете. Логика построения аналогична "Критерии фильтрации"
    filters array Выражение, может содержать в себе как простые фильтры, так и дерево фильтров
    field string Поле сущности к которой будет применяться фильтрация (список заранее определён для метода
    operator enum Оператор фильтрации. Список всех операторов можно получить в разделе "Операторы фильтрации"
    value string Значение для оператора фильтрации. Необязательное поле, если оно отсутствует, то считается пустота.
    Сортировка применяемая в отчете.
    sort array Сортировка применяемая в отчете. Логика построения аналогична "Сортировка данных"
    Параметры столбцов
    column_params object нет Список столбцов с параметрами.
    field object да Мнемоника или идентификатор столбца параметры которого изменены
    widths number да Ширина столбца. Задается в px.

    JSON структура запроса

    {
    "id": "number",
    "jsonrpc": "2.0",
    "method": "getobj.report_state",
    "params": {
        "access_token": "string",
        "user_id": "number",
        "report_id": "number"
    }
}
